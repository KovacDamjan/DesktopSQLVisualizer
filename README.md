# DesktopSQLVisualizer

Desktop SQL client built with Angular + Electron.

## Features

- Desktop app shell (Electron) + Angular frontend
- Supported engines: MSSQL, MySQL, PostgreSQL, DB2
- Save/load connection profiles from JSON config file
- Test connection and load database/object metadata
- Actionable error hints for connection and SQL failures
- Monaco SQL editor (format, params detection, execute/explain actions)
- Parameter panel controls (show/hide params, parameter presets, metadata-aware types)
- Context-aware SQL autocomplete (keywords, tables/views, functions/procedures, params, columns)
- AG Grid results with multi-result-set tabs, sorting, filtering and search highlight
- SQL execute mode (selection or full editor) + SQL explain mode
- Sequential multi-statement execution with multi-result-set aggregation
- Cancel running SQL execution from toolbar
- Max rows limit before execute (default `1000`)
- Export all result sets to one `.xlsx` file (`Result_1`, `Result_2`, ...)
- Favorites + recent history per connection
- Inline result editing with PK detection (`connection:list-primary-keys`)
- PostgreSQL procedure/cursor handling and DB2 safe query mode
- ER diagram with draggable tables, auto-layout, schema/search/focus, and large-model safety limits

## Requirements

- Node.js 20+ (recommended)
- npm

For DB2 support (`ibm_db`), native addon rebuild may be needed after install:

```bash
npm run rebuild:ibmdb
```

`rebuild:ibmdb` is cross-platform (`macOS`, `Linux`, `Windows`).
On `Windows`, the script will also attempt automated prerequisite install (`Python` + `VS Build Tools`) via `winget` if missing.
It also runs `ibm_db` driver repair (`installer/driverInstall.js`) and verifies DB2 CLI DLL presence.

## Development

Install dependencies:

```bash
npm install
```

Run Angular in browser mode:

```bash
npm start
```

Run desktop app in development (Angular dev server + Electron):

```bash
npm run desktop:dev
```

## Build

Build Angular bundle:

```bash
npm run build
```

Build desktop package with `electron-builder`:

```bash
npm run desktop:build
```

## App Updates

- App now supports desktop update flow (`Check updates`, `Download update`, `Install update`) in the top bar.
- Auto-update works only in packaged app (not `desktop:dev`).
- This project is prepared for GitHub Releases auto-update via `package.json -> build.publish`.
- Without a publish provider, update checks will return an informative error message in-app.

### GitHub Release Publish

Set environment variables before publishing:

```bash
GH_OWNER=<github-owner-or-org>
GH_REPO=<github-repo-name>
GH_TOKEN=<github-token-with-repo-access>
PRIVATE_GIT_REMOTE=<private-git-remote-name, default: origin>
PRIVATE_GIT_USERNAME=<optional-private-git-username>
PRIVATE_GIT_PASSWORD=<optional-private-git-password>
```

Use your existing `.env` file and set/update those variables there.

Then publish:

```bash
npm run desktop:publish:github
```

Publish fix/rebuild artifacts to the current version tag (no auto-bump):

```bash
npm run desktop:publish:github:current-tag
```

Notes:
- `GH_TOKEN` is required for creating/updating GitHub Release assets.
- Auto-update clients use release metadata files generated by `electron-builder` (for example `latest.yml`).
- Publish script performs preflight checks before build:
  - token validity (`/user`) and repo access (`/repos/$GH_OWNER/$GH_REPO`)
  - private git working tree must be clean (`git status --porcelain` must be empty)
  - current branch must be in sync with `PRIVATE_GIT_REMOTE/<branch>` (ahead/behind = 0)
  - git tags are refreshed from `PRIVATE_GIT_REMOTE` before release note calculation
- On successful preflight/build, script creates a new local git tag `v<package.json version>`.
- If tag `v<version>` already exists locally, script auto-bumps patch version in `package.json` until a free tag is found.
- Existing tag for the same version is reused when current platform assets are still missing (example: mac release exists, Windows publish adds Windows assets to same version).
- `desktop:publish:github:current-tag` forces publish to `v<current package.json version>` (uses env `PUBLISH_CURRENT_TAG=1`).
- After auto-bump, script automatically creates commit `New version X.Y.Z` (with change summary) and pushes it to `PRIVATE_GIT_REMOTE/<branch>`.
- Script validates version consistency before tagging (`package.json` must match target release version), otherwise publish is aborted.

## Editor Commands

In Monaco editor:

- `Ctrl/Cmd + Enter` -> Execute
- `Ctrl/Cmd + Shift + Enter` -> Explain
- `Ctrl/Cmd + Shift + P` or `F1` -> SQL command palette flow (`>sql`)
- Right click in editor -> context menu actions (`Execute`, `Explain`, `Format SQL`, ...)

## Parameter Handling

- `Params (N)` detects placeholders and builds parameter inputs.
- `Hide Params` lets you work directly in SQL editor without parameter panel.
- Parameter panel visibility is persisted between app restarts.
- Parameter presets support save/load/delete for current SQL signature.
- DB2 supports array parameter handling and routine metadata-based parameter typing.

## Autocomplete Notes

- Trigger suggestions with typing or `Ctrl/Cmd + Space`.
- If SQL contains `SELECT ...` (without `FROM` yet), function suggestions insert `schema.function(...)` (no duplicate `SELECT`).
- Schema-qualified typing (example `public.`) suggests object names without duplicating schema.

## ER Diagram Notes

- ER view keeps all tables visible.
- For performance on very large schemas, relation rendering is capped (tables remain visible).

## Connection Config File

Connections are stored in:

- `connections.config.json` in Electron `app.getPath('userData')`

In the app, the active path is shown under `Connections -> Config file`.

## Main Backend Entry

Core desktop/backend logic is in:

- `electron-main.cjs`

This file defines Electron IPC handlers for:

- connection management (`connections:*`, `connection:*`)
- SQL execution and explain (`sql:execute`, `sql:explain`)
- metadata/object helpers (`connection:list-objects`, `connection:list-primary-keys`)
